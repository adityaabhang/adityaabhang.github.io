Overview

This project is an industry collaboration with TDS to develop an in-house real-time monitoring system for their CNC plasma cutting machine. The core objective is to classify the thickness of sheet material being cut by analyzing acoustic emission signatures and current draw patterns during operation — without any manual measurement or interruption to the cutting process.

By mapping sensor data to material thickness, the system provides operators with live validation that the correct material is being processed, reducing scrap rates and catching material mix-ups before they result in defective parts. The entire monitoring pipeline feeds into Grafana dashboards for real-time visualization and historical analysis.

Methodology

1. Sensor Instrumentation: Acoustic sensors and current sensors have been installed on the plasma cutting machine. The acoustic sensors capture the sound profile of the plasma arc during cutting, which varies predictably with material thickness — thicker material produces a distinct acoustic signature compared to thin sheet. Current sensors measure the power draw of the cutting head, which also correlates with material thickness and cutting speed.

2. Data Acquisition: A Raspberry Pi serves as the edge data collection node, continuously sampling both sensor streams and logging timestamped readings. The Pi handles data preprocessing, buffering, and transmission to the monitoring backend.

3. Thickness Classification: The current approach uses threshold matching on acoustic and current features to classify material thickness. The sensor signatures for different thicknesses are sufficiently distinct that rule-based thresholds achieve reliable classification. An ML model remains as a backup option if edge cases or noise conditions require more sophisticated pattern recognition.

4. Visualization & Alerting: All sensor data and classification results flow into Grafana dashboards, providing operators with real-time thickness readings, historical trends, and anomaly alerts when sensor readings fall outside expected ranges for the programmed material.

Challenges

Industrial noise environment: A CNC plasma cutting shop is acoustically hostile. Isolating the relevant acoustic signature of the plasma arc from ambient shop noise, other machines running simultaneously, and material handling sounds requires careful sensor placement and signal conditioning. We're working on filtering techniques to extract the signal-of-interest from the noise floor.

Threshold robustness across conditions: While threshold matching works well under controlled conditions, variations in consumable wear (electrode and nozzle degradation), gas pressure fluctuations, and environmental temperature changes can shift the baseline sensor readings. The system needs to be robust enough to handle these drift conditions without producing false classifications.

Real-time reliability: The monitoring system must run continuously during production without drops or latency. The Raspberry Pi is handling data acquisition, processing, and transmission simultaneously, so the pipeline needs to be efficient enough to maintain consistent sampling rates under load.
