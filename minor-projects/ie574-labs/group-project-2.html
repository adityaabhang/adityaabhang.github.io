Overview

This project put the complete imitation learning pipeline into practice on a real robot. Using the SO-ARM101 dual-arm teleoperation platform and HuggingFace's LeRobot framework, the team collected human-demonstrated pick-and-place trajectories, trained a visuomotor policy on those demonstrations, and deployed the learned policy autonomously on the physical follower arm.

Rather than hand-coding motion sequences or writing explicit control logic, the robot learned the task entirely from observation — mapping camera images and joint states directly to control actions via a trained neural network.

Full pipeline: Calibrate motors → Teleoperate and record demonstrations → Train visuomotor policy with LeRobot → Run inference on the real robot → Evaluate pick-and-place performance.

The Platform: SO-ARM101

The SO-ARM101 is a leader–follower dual-arm teleoperation system. The human operator directly moves the leader arm; its joint positions stream in real time to the follower arm, which mirrors every motion precisely. This architecture makes it natural to collect high-quality, physically realistic manipulation demonstrations for imitation learning.

The platform includes 3D-printed structural parts, Feetech serial bus servo motors, a motor control board, USB communication cables, table clamps, and an RGB camera for visual observations during training and inference. Group 7's dedicated unit was located in DUDL 3443 with access granted for a two-week window.

Methodology

Step 1 — Motor Calibration: All servo joints on both leader and follower arms were calibrated using LeRobot's built-in calibration routines, ensuring accurate position measurement and command execution across the full range of motion.

Step 2 — Teleoperation Setup: Configured real-time leader-to-follower joint mirroring over USB. Practiced the pick-and-place task manually to validate responsiveness and confirm the arm geometry was consistent with the target object position.

Step 3 — Dataset Collection: Recorded multiple demonstration episodes using LeRobot's dataset pipeline. Each episode captured synchronized RGB camera frames and robot joint state readings as the operator guided the leader arm through the pick-and-place motion.

Step 4 — Policy Training: Trained a visuomotor imitation learning policy in PyTorch using the recorded dataset. Weights and Biases was used for experiment tracking. The policy learned to map visual observations and robot state directly to joint control commands.

Step 5 — Inference and Evaluation: Deployed the trained policy on the follower arm for autonomous execution. The robot predicted joint actions from camera input at each timestep, executing the pick-and-place task without any teleoperation input.

Challenges

Demonstration quality directly determines policy quality: Unlike simulation, real-robot demonstrations are not repeatable by replay. Jerky or inconsistent operator motions produced noisy training data that directly degraded policy performance. Consistent, smooth demonstrations required deliberate practice before recording.

Physical variability: Minor differences in object placement, lighting, and camera angle between training and test time caused inference failures. This highlighted how sensitive visuomotor policies are to distribution shift — a challenge that simulation environments typically hide.

Hardware reliability: Servo calibration drift, cable tension affecting joint positions, and USB communication latency introduced subtle inconsistencies. These required hardware-level debugging that would not appear in a simulated experiment.
