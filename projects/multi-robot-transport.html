Cooperative Transport Planning for Multi-Robot Systems

Oct – Dec 2025 • West Lafayette, IN • Completed
Reinforcement Learning Multi-Agent Systems Python NumPy Dynamic Programming
Project hero image here (760×360)

Overview

This project tackled the classic "piano movers" problem — how do you coordinate two robots carrying a shared payload through a cluttered home environment? The twist: the robots must maintain a fixed inter-agent distance throughout the transport, meaning they can't independently navigate around obstacles. Every move by one robot constrains the other.

The goal was to benchmark model-based dynamic programming against model-free reinforcement learning for this constrained cooperative navigation task, analyzing convergence speed, path optimality, and computational cost across three algorithms: Policy Iteration, Value Iteration, and Q-Learning.

Methodology

1. Environment Design: The environment was modeled as a static occupancy grid representing a home floor plan, parsed from DXF architectural drawings. The grid captured walls, furniture, doorways, and narrow passages. Two agents were placed in this grid with a rigid distance constraint — they could only move in tandem, maintaining exact separation.

2. Algorithm Implementation: Three algorithms were implemented and benchmarked against each other:

  • Policy Iteration: Full model-based dynamic programming. Iteratively evaluates and improves a policy until convergence. Guarantees optimality but requires complete environment knowledge and is computationally expensive in large state spaces.
  • Value Iteration: Another model-based approach that directly computes optimal values through iterative Bellman updates. Faster convergence than Policy Iteration in practice, but still requires the full transition model.
  • Q-Learning: Model-free RL approach. Learns action-value functions purely from experience without needing an environment model. More adaptable to unknown environments but sample-inefficient and slower to converge.

3. Constraint Handling: The kinematic constraint (fixed inter-agent distance) was enforced directly in the state-action space. Any joint action that would violate the distance constraint was masked as invalid, effectively pruning the action space to only physically feasible moves.

Challenges

State space explosion: With two agents on a grid, each with position coordinates, the joint state space grows quadratically. Adding the fixed-distance constraint actually helps by pruning many joint states, but the remaining valid state space is still large enough to make convergence slow — especially for Q-Learning, which needed significantly more episodes to reach reasonable policies.

Narrow passage coordination: The hardest scenarios involved doorways and tight corridors. The rigid distance constraint meant both agents had to fit through simultaneously, often requiring non-intuitive routing detours that simple heuristics would miss. The DP methods found these optimal detours; Q-Learning took much longer to discover them.

Comparing fairly across paradigms: Model-based and model-free methods have fundamentally different computational profiles — wall-clock time, sample efficiency, and memory usage don't scale the same way. Designing metrics that gave a fair, insightful comparison rather than just declaring one "better" required careful thought about what matters in different deployment scenarios.

Trade-offs
Quantified cost of model-free adaptability vs. model-based optimality

The analysis demonstrated that model-based approaches (Policy and Value Iteration) consistently found optimal or near-optimal paths but required full environment knowledge and significant computation. Q-Learning produced reasonable but suboptimal paths with much less environment knowledge, highlighting the classic exploration-exploitation trade-off in constrained multi-agent settings.
