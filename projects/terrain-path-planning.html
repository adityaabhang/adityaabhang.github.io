Vision-Guided Terrain-Aware Path Planning

Oct – Dec 2025 • West Lafayette, IN • Completed
Gazebo ROS2 DeepLabV3+ LiDAR RGB-D A* RRT* Python
Project hero image here (760×360)

Overview

This project built a complete closed-loop simulation pipeline for autonomous ground robot navigation that goes beyond simple obstacle avoidance. Instead of treating all non-occupied space as equally traversable, the system uses semantic understanding of terrain to plan paths that are not just collision-free but terrain-optimal — preferring smooth ground, avoiding soft soil, and routing around vegetation even when technically passable.

The pipeline fuses LiDAR range data with RGB-D camera feeds, runs real-time semantic segmentation to classify terrain types, and injects these classifications into a dynamic costmap that feeds the navigation planner. The result is a robot that makes smarter routing decisions based on what the ground actually is, not just where obstacles are.

Methodology

1. Sensor Fusion Pipeline: The simulated robot in Gazebo was equipped with both a LiDAR scanner and an RGB-D camera. LiDAR provided precise geometric obstacle detection — walls, rocks, large objects. The RGB-D camera provided color and depth data that the segmentation model could process. Both streams were synchronized and fused into a unified perception layer.

2. Semantic Segmentation: A DeepLabV3+ model was deployed to classify every pixel in the RGB-D stream into terrain categories: drivable (concrete, asphalt, packed dirt), cautious (gravel, grass, uneven ground), and non-drivable (deep mud, water, dense vegetation). These per-pixel classifications were projected into the robot's local coordinate frame using depth data.

3. Dynamic Costmap Generation: The terrain classifications were integrated into ROS2's costmap framework. Each terrain type was assigned a traversal cost — drivable terrain had low cost, cautious terrain had elevated cost, and non-drivable was treated as an obstacle. This costmap updated in real-time as new sensor data arrived, allowing the planner to react to previously unseen terrain.

4. Path Planning & Benchmarking: Two planners were tested against this semantically-enriched costmap: A* (graph search, guarantees optimality on the discretized grid) and RRT* (sampling-based, asymptotically optimal, better in high-dimensional spaces). Both were benchmarked against standard implementations that used only binary occupancy (obstacle / free space) to quantify the improvement from terrain awareness.

Challenges

Segmentation inference latency: DeepLabV3+ is not the lightest model. Running it at camera frame rate while simultaneously updating the costmap and planning paths required careful optimization of the inference pipeline. We used model quantization and selective inference (not every frame needed full segmentation) to keep the loop rate acceptable for real-time navigation.

Costmap boundary effects: Where terrain types met — the edge between concrete and grass, for example — the segmentation model sometimes produced noisy classifications. These noisy boundaries in the costmap created jittery planned paths that oscillated between terrain types. Spatial smoothing and temporal filtering on the costmap helped stabilize the planner's behavior.

Sim-to-real gap considerations: While this was a simulation project, we designed the pipeline with eventual real-world deployment in mind. The Gazebo world was textured with realistic terrain appearances, and the segmentation model was trained on mixed synthetic and real-world data to improve eventual transfer.

Outcomes

Improved
Measurable improvement in obstacle avoidance vs. standard A* and RRT* baselines
Real-time
Dynamic costmap updates with semantic terrain classification at navigation rate
2 Planners
A* and RRT* benchmarked with and without terrain awareness

The terrain-aware planners consistently chose safer, more efficient routes compared to binary-occupancy baselines — particularly in environments with deceptive terrain (areas that looked passable geometrically but were actually soft ground or dense vegetation). The semantic costmap added minimal computational overhead while significantly improving navigation quality.
